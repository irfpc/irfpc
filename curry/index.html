مفهوم curry در Functional Programming

April 06, 2020
محمد حسنی

قانون curry یعنی هر function در هر application (همان call کردن) صرفا اجازه داره حداکثر یک argument را به عنوان ورودی دریافت کنه. چه اهمیتی داره؟ باعث تفاوت بین دو قطعه کد بالا میشه. نسخه uncurriedAdd دوتا argument را به عنوان ورودی دریافت میکنه (پس قانون curry را رعایت نمیکنه). این موضوع باعث شده که که uncurriedAdd با map قابل ترکیب نباشه و برای استفاده از آن داخل map مجبوریم از یک function واسط استفاده کنیم.

نسخه curriedAdd اما همانطور که میبینید به صورت نصف و نیمه apply شده (partial application) و خیلی راحت و بدون فانکشن واسط با map ترکیب شده.
